# Работа с Figma API

Основные технические ограничения, методы и нюансы работы с ID нод.

## Лимиты (Rate Limits)
- Figma накладывает ограничения на количество запросов в минуту. 
- **Важно**: Нужно использовать заголовок `X-RateLimit-Limit` и `Retry-After` для обработки очередей.
- При получении 429 ошибки — экспоненциальная задержка.

## Основные Endpoint-ы
1. `GET /v1/teams/:team_id/projects` — список проектов команды.
2. `GET /v1/projects/:project_id/files` — список файлов в проекте.
3. `GET /v1/files/:key` — получение полной структуры файла.
4. `GET /v1/files/:key/components` — метаданные компонентов.

## Особенности JSON структуры
- Файлы могут быть огромными (сотни мегабайт).
- Рекомендуется использовать параметры `depth` или `ids` в запросе, если это возможно, но для полной индексации придется качать всё.

## Форматы Node ID

### Обычные ноды
Стандартный формат: `pageIndex:nodeIndex` (например `42155:6`).
В URL используется дефис вместо двоеточия: `node-id=42155-6`.

### Ноды внутри инстансов (compound ID)
REST API возвращает составные ID для детей инстансов:
```
I16259:10540;7:3093;7:3058;7:3040
```

Структура:
- **`I`** — префикс, означает что нода внутри инстанса
- **`16259:10540`** — ID родительского инстанса на холсте
- **`;`** — разделитель уровней вложенности
- **`7:3040`** — ID элемента в мастер-компоненте

### ⚠️ Ограничение REST API
REST API **не возвращает canvas-level ID** для детей инстансов. Когда пользователь кликает на элемент внутри инстанса в Figma UI, приложение генерирует "абсолютный" ID (например `42155:6`), который доступен только через:
- **Plugin API** (`figma.getNodeByIdAsync()`)
- **MCP** (Figma Desktop App)

REST API такие ID не предоставляет ни в дереве нод, ни в `components` метаданных.

### Стратегия линковки (текущая реализация)
Поскольку "настоящие" ID детей инстансов недоступны через REST API:

| Тип ноды | ID в БД | Что берём для URL | Результат |
|---|---|---|---|
| Обычная нода | `42155:6` | Как есть → `42155-6` | ✅ Точный переход |
| Инстанс (top-level) | `16259:10540` | Как есть → `16259-10540` | ✅ Переход на инстанс |
| Ребёнок инстанса | `I16259:10540;7:3093` | Первый сегмент → `16259-10540` | ⚠️ Переход на родительский инстанс |

Реализация: [`apps/web/src/utils/figmaUtils.ts`](../apps/web/src/utils/figmaUtils.ts) — функция `formatNodeIdForUrl`.

### Возможное улучшение
Для точных deep links на детей инстансов потребуется интеграция с Figma Plugin API, который может вернуть canvas-level ID через `figma.getNodeByIdAsync()`.
