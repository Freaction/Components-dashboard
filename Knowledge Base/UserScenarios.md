# Сценарии использования (User Scenarios)

Основная целевая аудитория проекта — **дизайнеры**. Инструмент помогает им анализировать базу компонентов всех команд для ускорения рефакторинга дизайн-системы.

## Основные задачи
1. **Анализ покрытия дизайн-системы**: Поиск всех используемых компонентов во всех проектах компании для понимания их популярности и критичности.
2. **Поиск и устранение "Детачей"**: Быстрое нахождение элементов, которые были отсоединены от мастер-компонентов, для их последующего возврата в систему.
3. **Быстрая навигация**: Использование **Deep Links** для мгновенного перехода из Дашборда на конкретную страницу и слой в Figma, где обнаружена проблема.
4. **Консолидация кастома**: Нахождение уникальных, часто повторяющихся решений (кастомных элементов), которые еще не стали частью общей библиотеки, для их стандартизации.

## Примеры использования
- *“Я хочу найти все кнопки в проекте 'Mobile App', которые не привязаны к основному компоненту Button, чтобы исправить их за один раз.”*
- *“Мне нужно понять, какие команды чаще всего создают свои локальные компоненты вместо использования глобальных, чтобы провести с ними обучение.”*
- *“Я ищу все места, где используется старый стиль иконки поиска, чтобы заменить его на новый через прямые ссылки на макеты.”*
