# Масштабирование поиска и навигация

Когда в системе проиндексированы десятки команд и тысячи файлов, количество элементов (нод) исчисляется миллионами. Просто вывести их списком невозможно.

## 1. Проблема: "Тысячи элементов в поиске"
Как отображать результаты, чтобы дизайнер не утонул в данных:

### Агрегация и Группировка
- **Группировка по мастер-компоненту**: Если найдено 500 кнопок, мы показываем одну карточку "Button" с пометкой "500 использований". При клике раскрывается список конкретных мест.
- **Группировка по командам/проектам**: Сначала показываем, в каких проектах найдено больше всего совпадений.
- **Умные фильтры**: По умолчанию скрывать "мусор" (фильтрация шума) и показывать только элементы на "чистых" страницах.

### Техническая оптимизация UI
- **Virtual Scrolling**: Рендеринг только тех карточек, которые видны на экране.
- **Infinite Scroll / Pagination**: Подгрузка данных порциями по 50-100 элементов.
- **Skeleton Screens**: Показ заглушек во время подгрузки тяжелых данных (например, превью).

## 2. Deep Links

### Зачем нужны Deep Links?
Просто ссылка на страницу (`https://figma.com/file/:key/:name`) заставляет дизайнера вручную искать нужный слой среди сотен фреймов. Deep Link (`?node-id=:id`) выделяет конкретный объект и зумится на него. Для рефакторинга это экономит до 30-60 секунд на каждом элементе.

### Формирование ссылок (client-side)
Ссылки формируются на фронтенде без обращения к Figma API. Вся логика сосредоточена в [`figmaUtils.ts`](../apps/web/src/utils/figmaUtils.ts).

**Алгоритм `formatNodeIdForUrl`:**
1. Убираем префикс `I` (инстансы)
2. Для составных ID (с `;`) берём **первый сегмент** — это родительский инстанс
3. Заменяем `:` на `-` для URL-формата

```typescript
// "I16259:10540;7:3093;7:3040" → "16259-10540"
// "42155:6" → "42155-6"
```

### Ограничение: дети инстансов
REST API не возвращает canvas-level ID для элементов внутри инстансов. Подробнее — см. [[FigmaAPI|Работа с Figma API]], раздел «Форматы Node ID».

Текущее поведение: ссылка ведёт на ближайший родительский инстанс.

### Использование в компонентах
- [`NodesView.tsx`](../apps/web/src/views/NodesView.tsx) — передаёт `node.id` в `generateFigmaLink`
- [`TeamsView.tsx`](../apps/web/src/views/TeamsView.tsx) — аналогично, через `getFigmaLink`

Оба компонента используют `node.id` напрямую; `formatNodeIdForUrl` внутри `generateFigmaLink` обрабатывает все случаи автоматически.
