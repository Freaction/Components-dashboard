# План реализации (v2.0)

## Фаза 1: Окружение и Монорепозиторий (DONE)
1. Инициализация проекта через `bun init`.
2. Настройка единого запуска (Frontend + Backend) через `bun --filter`.
3. Настройка Drizzle ORM для SQLite.

## Фаза 2: Быстрый Парсинг (ETL) - DONE
1. Интеграция с Figma API (axios stream + batch insert).
2. Оптимизация БД: Включение SQLite FTS5 для поиска.
3. Аналитическая модель: Добавление fingerprint и depth.
4. Модульная архитектура: Разделение на core/modules.

## Фаза 3: Аналитика и Поиск - В ПРОЦЕССЕ
1. Реализация полнотекстового поиска через API (MATCH).
2. Расчет Health Score (внедрение логики анализа стилей).
3. Detach Analysis (сравнение по fingerprint).

## Фаза 4: UI и Real-time
1. Разработка Dashboard на React.
2. Настройка Eden Treaty для мгновенной синхронизации типов между бэкендом и фронтендом.

## Фаза 5: Тестирование и Валидация
1. **Тест-драйв парсера**: Загрузка тестового JSON-файла (dump Figma API) и проверка корректности записи в DuckDB.
2. **Benchmark поиска**: Замер скорости FTS5/DuckDB на выборке в 100,000 нод.
3. **End-to-End**: Отображение данных из DuckDB на фронтенде в реальном времени.
